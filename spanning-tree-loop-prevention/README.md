# Spanning Tree Loop Prevention Lab

## Objective
Explore how the Spanning Tree Protocol (STP) prevents switching loops in redundant network topologies.

## Skills Demonstrated

- STP behavior analysis
- Layer 2 loop prevention
- Network topology troubleshooting

## Tools Used

Cisco Packet Tracer

## Key Concepts

Layer 2 switching loops can cause broadcast storms and network instability. The Spanning Tree Protocol prevents loops by placing redundant links into a blocking state.

## Validation

Network stability and STP behavior were verified using:

- spanning-tree status commands
- connectivity testing
- topology observation
